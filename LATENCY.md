```text
버전: 1.4.31
BACK 버튼을 설정된 실제 이전 시간 표시로 변경
LAT 버튼을 한국어 저지연 상태 표시로 변경
저지연 상태를 저지연/저지연↑/저지연!/저지연Ⅱ로 구분
기능 그룹 간격 분리
설정 톱니바퀴를 SVG 아이콘 버튼으로 통일
저지연 톱니바퀴 클릭 시 LAT 전용 모드 메뉴 표시
숨김 버튼 실행 시 복구 경로 안내
버튼 높이/글자 크기/focus-visible 보강
hover와 활성 상태 색상 분리
자동/안정 모드는 화질 보호를 위해 live edge에서 조금 더 떨어져 보정
앞쪽 버퍼가 0.9초 미만이면 배속/seek 보정을 멈춤
자동/안정 모드 배속 상한을 1.04x로 낮추고 빠른 반응 모드만 1.06x 유지
seek 목표 지점 뒤에 0.5초 buffered 여유를 요구
저지연 자동/빠른 반응 모드에서 최고 화질 선택을 주기적으로 시도
팝업 고급 상태에 화질 선택 시도/성공 횟수 표시
```

## 전체 판정

**반복 사용자는 편하지만, 처음 보는 사용자에게는 의미가 압축돼 있습니다.**
외형은 깔끔하고 CHZZK의 어두운 화면에도 잘 어울립니다. 다만 `LAT`, `BACK`, 파란색 상태의 의미는 설명 없이 바로 이해하기 어렵습니다.

* 기존 사용자 기준: **8/10**
* 처음 설치한 사용자 기준: **5~6/10**

## 가장 먼저 바꿀 세 가지

### 1. `BACK`은 실제 시간으로 표시하는 게 낫습니다

`BACK`은 브라우저 뒤로가기나 영상 되감기로 오해할 수 있습니다. 실제 기능은 “설정된 과거 프레임 캡처”이므로 다음이 훨씬 직관적입니다.

```text
BACK → -0.5s
BACK → -1.0s
```

권장 배치:

```text
[SHOT] [-0.5s] [REC]
```

설정값이 바뀌면 버튼 표시도 함께 바꾸면 사용자가 현재 설정을 옵션 페이지까지 들어가지 않고 확인할 수 있습니다.

---

### 2. 파란 `LAT` 상태는 지금 의미가 전달되지 않습니다

현재 코드상 LAT는 다음처럼 상태별 색상이 다릅니다.

```text
일반 켜짐: 초록
따라잡는 중: 파랑
안정화 중: 주황
사용자 이동으로 일시 중지: 회색
```

그런데 버튼 글자는 계속 `LAT`입니다. 따라서 스크린샷의 파란색이 코드상 “배속으로 따라잡는 중”이어도, 일반 사용자는 그냥 “켜짐”으로 인식합니다. 네 가지 색 의미를 사용자가 외워야 하는 구조입니다. ([GitHub][1])

다음처럼 글자나 기호도 같이 바꾸는 편이 낫습니다.

```text
LAT     정상 작동
LAT↑    따라잡는 중
LAT!    버퍼 안정화 중
LATⅡ    사용자 이동으로 일시 중지
```

버튼 폭은 고정한 채 작은 기호만 바꾸면 레이아웃 이동도 거의 없습니다.

`REC`가 녹화 중에 빨간색뿐 아니라 `STOP`으로 글자까지 바뀌는 방식은 좋습니다. LAT도 같은 원칙을 적용하는 게 맞습니다. ([GitHub][1])

또한 현재 CSS는 일반 버튼의 `hover`와 LAT의 일반 활성 상태가 모두 초록 계열입니다. hover는 배경을 완전히 바꾸기보다 테두리나 밝기만 조절하고, 활성 상태 색은 별도로 유지하는 편이 상태 구분에 유리합니다. ([GitHub][2])

---

### 3. 기능별 그룹을 시각적으로 분리해야 합니다

현재 모든 버튼 간격이 동일한 6px라서 LAT 설정, 캡처 기능, 화면 숨기기가 전부 같은 그룹처럼 보입니다. ([GitHub][2])

다음 구조를 권합니다.

```text
[LAT][⚙]   [SHOT][-0.5s][REC]   [눈 가림]
```

```text
LAT ↔ 설정: 4px
캡처 버튼끼리: 4~6px
그룹 사이: 10~12px
```

구분선까지 넣을 필요는 없습니다. 간격만 달라도 충분합니다.

## 세부적으로 보이는 문제

### 톱니바퀴 크기가 다른 아이콘과 맞지 않습니다

코드에서 설정 버튼은 `⚙` 문자로 만든 일반 42px 버튼이고, 눈 가림 버튼만 별도의 SVG 아이콘과 34px 아이콘 폭을 사용합니다. 그래서 화면에서도 톱니바퀴는 상대적으로 작고, 버튼 자체는 불필요하게 넓어 보입니다. ([GitHub][1])

다음처럼 통일하는 게 낫습니다.

```text
설정 버튼: 34×32 또는 34×34
톱니바퀴: 16~17px SVG
눈 가림: 16~17px SVG
```

또한 `[LAT][⚙]`를 테두리가 이어진 분할 버튼처럼 만들면 톱니바퀴가 전체 설정이 아니라 LAT 설정이라는 점이 더 명확해집니다.

---

### 설정 버튼이 곧바로 전체 옵션 페이지를 여는 것은 조금 무겁습니다

현재 LAT 옆 톱니바퀴는 전체 확장 옵션 페이지를 엽니다. 반면 확장 팝업에는 이미 `자동`, `빠른 반응`, `안정 우선`, `직접 설정` 모드가 마련돼 있습니다. ([GitHub][1])

사용자 기대에 더 맞는 동작은 다음입니다.

```text
톱니바퀴 클릭
-> 작은 LAT 전용 메뉴

자동(권장)
빠른 반응
안정 우선
고급 설정 열기
```

전체 옵션 페이지를 계속 열 예정이라면 적어도 LAT 설정 영역으로 바로 스크롤되게 해야 합니다.

---

### 눈 가림 버튼은 복구 방법을 알려줘야 합니다

아이콘 자체는 적절합니다. 다만 잘못 누르면 도구 모음 전체가 사라지므로 다른 버튼보다 약간 떨어뜨려 두는 것이 안전합니다.

숨길 때 메시지도 다음처럼 구체적으로 보여주는 편이 좋습니다.

```text
버튼을 숨겼습니다.
다시 표시: 확장 아이콘 → 화면 버튼 표시
```

팝업에는 이미 `화면 버튼 표시` 옵션이 있으므로 이 경로를 안내하면 됩니다. ([GitHub][3])

---

### 크기는 마우스 사용에는 괜찮지만 글자가 조금 작습니다

현재 버튼은 기본적으로 `42×32px`, 글자는 `11px`입니다. 컴팩트하다는 장점은 있지만, 고해상도 모니터나 영상에서 시선을 빠르게 옮길 때는 조금 빡빡합니다. ([GitHub][2])

권장값:

```css
height: 34px;       /* 여유가 있으면 36px */
font-size: 12px;
font-family: system-ui, sans-serif;
```

폭은 지금처럼 42px를 유지해도 됩니다. 키보드 사용자를 위해 명확한 `:focus-visible` 테두리도 추가하는 편이 좋습니다.

## 최종 권장 형태

```text
[LAT↑][⚙]   [SHOT][-0.5s][REC]   [◉̸]
```

상태 규칙:

```text
꺼짐       어두운 배경
켜짐       파란 배경
따라잡기   파란 배경 + ↑
안정화     주황 배경 + !
일시중지   회색 배경 + Ⅱ
녹화       빨간 배경 + STOP
```

가장 효과가 큰 수정 순서는 **`BACK`을 시간 표시로 변경 → LAT 상태를 색상 외 기호로도 표시 → 기능 그룹 간격 분리**입니다. 현재 디자인을 크게 뜯어고칠 필요는 없습니다.

아니요. 한글 사용자 기준으로는 **처음 봤을 때 절반 정도만 의미를 추측할 수 있습니다.**

| 현재     | 첫인상                        | 권장                  |
| ------ | -------------------------- | ------------------- |
| `LAT`  | 거의 이해하기 어려움                | `저지연`               |
| ⚙      | 설정으로 이해 가능                 | 그대로 유지              |
| `SHOT` | 캡처로 추측 가능하지만 즉각적이지 않음      | `캡처`                |
| `BACK` | 브라우저 뒤로가기·영상 되감기로 오해       | `-1초`처럼 실제 값 표시     |
| `REC`  | 비교적 익숙함                    | `녹화`, 녹화 중에는 `중지`   |
| 눈 가림   | 숨김으로 추측 가능하지만 무엇을 숨기는지 불명확 | 아이콘 유지 + `버튼 숨김` 툴팁 |

가장 직관적인 형태는 다음입니다.

```text
[저지연] [⚙]   [캡처] [-1초] [녹화]   [눈 가림]
```

공간이 부족하다면 최소한 이렇게 바꾸는 게 좋습니다.

```text
[LAT] → [지연↓]
[SHOT] → [캡처]
[BACK] → [-1초]
[REC] → [REC]
```

특히 **`BACK`은 반드시 바꾸는 편이 낫습니다.** 한글 사용자뿐 아니라 영어 사용자도 뒤로가기 버튼으로 받아들일 가능성이 큽니다.

그리고 파란색 `LAT`는 사용자가 거의 확실히 **“저지연 기능이 켜졌다”**고 이해합니다. 실제로 파란색이 “따라잡는 중” 상태라면 전달되지 않습니다. 상태를 구분하려면 다음처럼 글자도 바뀌어야 합니다.

```text
저지연       정상
저지연↑      따라잡는 중
저지연!      안정화 중
저지연Ⅱ      일시 중지
```

결론적으로, 한국어 사용자를 주 대상으로 한다면 영어 약어보다 **`저지연 / 캡처 / -1초 / 녹화`**가 훨씬 낫습니다. 버튼 폭이 조금 늘더라도 처음 설치한 사용자의 이해도 차이가 큽니다.

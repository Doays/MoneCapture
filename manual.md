# Chzzk capture 사용 설명서

기준 버전: 1.4.31

Chzzk capture는 CHZZK 라이브와 다시보기에서 현재 장면 캡처, 이전 장면 캡처, 연속 캡처, 녹화, 5초 탐색, 저지연 보정, 통나무 표시 및 자동 수령을 사용할 수 있게 해주는 Chromium 확장 프로그램입니다.

이 문서는 처음 설치하는 과정부터 각 기능의 사용법, 권장 설정, 문제 해결 방법까지 순서대로 설명합니다.

---

## 1. 지원 환경

확장 프로그램은 Chrome, Microsoft Edge, Whale 등 Chromium 기반 브라우저를 기준으로 제작되었습니다.

기능이 적용되는 주소는 다음과 같습니다.

- CHZZK 라이브 방송 페이지
- CHZZK 다시보기 및 동영상 페이지

일반 CHZZK 홈 화면이나 다른 사이트에서는 화면 버튼이 나타나지 않습니다. 저장 폴더 직접 선택 기능은 브라우저의 파일 시스템 지원 여부에 따라 사용할 수 없을 수 있습니다.

---

## 2. 설치 방법

1. GitHub Releases에서 배포용 확장 프로그램 ZIP 파일을 받습니다.
2. 받은 ZIP 파일을 원하는 위치에 압축 해제합니다.
3. 브라우저 주소창에 `chrome://extensions`를 입력합니다.
4. 오른쪽 위의 `개발자 모드`를 켭니다.
5. `압축해제된 확장 프로그램을 로드합니다`를 누릅니다.
6. 압축을 푼 폴더 중 `manifest.json`이 들어 있는 폴더를 선택합니다.
7. 이미 열려 있던 CHZZK 페이지를 새로고침합니다.

GitHub가 자동으로 만드는 `Source code (zip)` 파일과 실제 배포용 ZIP은 다를 수 있습니다. 설치할 때는 배포자가 별도로 첨부한 확장 프로그램용 파일을 사용하는 것이 안전합니다.

---

## 3. 업데이트 방법

같은 폴더에 새 버전 파일을 덮어쓴 경우에는 다음 순서로 업데이트합니다.

1. `chrome://extensions`를 엽니다.
2. Chzzk capture 카드의 새로고침 버튼을 누릅니다.
3. 열려 있던 CHZZK 탭을 새로고침합니다.

새 폴더에 다시 압축을 풀어 설치하면 브라우저 환경에 따라 기존 설정이나 저장 폴더 권한이 이어지지 않을 수 있습니다. 가능하면 기존 설치 폴더에 새 파일을 덮어쓴 뒤 확장 프로그램만 새로고침하는 편이 편합니다.

---

## 4. 처음 설치한 뒤 할 일

브라우저 도구 모음에서 Chzzk capture 아이콘을 누르면 팝업이 열립니다.

처음에는 다음 항목만 확인하면 됩니다.

1. `화면 버튼 표시`가 켜져 있는지 확인합니다.
2. 저장 폴더를 직접 지정하려면 `저장 폴더 선택`을 누릅니다.
3. 라이브 방송을 자주 본다면 저지연 모드를 `자동`으로 둡니다.
4. 이전 장면 캡처 시간을 원하는 값으로 설정합니다. 기본값은 1초입니다.
5. CHZZK 라이브나 다시보기 페이지를 새로고침합니다.

저장 폴더를 지정하지 않아도 캡처와 녹화는 사용할 수 있습니다. 이 경우 브라우저의 기본 다운로드 방식으로 저장됩니다.

---

## 5. 화면 버튼 구성

라이브 방송에서는 보통 다음 순서로 버튼이 표시됩니다.

```text
[저지연] [설정]   [캡처] [-1초] [녹화]   [버튼 숨김]
```

다시보기에서는 라이브 전용인 저지연 버튼이 표시되지 않을 수 있습니다.

### 저지연

라이브 플레이어를 최신 재생 위치에 가깝게 유지합니다. 버튼 글자와 상태는 현재 동작에 따라 달라집니다.

- `저지연`: 기능이 켜져 있고 정상 상태
- `저지연↑`: 밀린 재생 위치를 따라잡는 중
- `저지연!`: 버퍼 상태가 좋지 않아 안정화 중
- `저지연Ⅱ`: 사용자가 재생 위치를 직접 옮겨 잠시 멈춘 상태

### 설정

저지연 모드를 빠르게 바꾸거나 전체 설정 화면을 엽니다.

### 캡처

현재 화면을 PNG 이미지로 저장합니다.

### -1초

설정된 시간만큼 이전 장면을 캡처합니다. 설정이 0.5초라면 `-0.5초`, 2초라면 `-2초`처럼 실제 값이 버튼에 표시됩니다.

### 녹화

현재 재생 중인 영상을 녹화합니다. 녹화가 시작되면 버튼이 `중지`로 바뀌며, 다시 누르면 파일을 저장하고 종료합니다.

### 버튼 숨김

화면 위 버튼을 숨깁니다. 다시 표시하려면 브라우저의 확장 프로그램 아이콘을 누른 뒤 `화면 버튼 표시`를 켭니다.

## 6. 현재 장면 캡처

현재 보고 있는 장면을 저장하는 방법은 세 가지입니다.

- 화면의 `캡처` 버튼 클릭
- 확장 프로그램 팝업의 즉시 캡처 버튼 클릭
- 기본 단축키 `S` 누르기

파일은 PNG 형식으로 저장됩니다. 파일명에는 촬영 시간이 들어갑니다.

```text
screenshot-YYYYMMDD-HHMMSS-mmm.png
```

영상 프레임을 직접 가져올 수 없는 상황에서는 브라우저가 현재 보이는 탭 화면을 기준으로 캡처할 수 있습니다. 이때 플레이어 버튼이나 마우스 포인터 주변 UI가 함께 찍히면 마우스를 영상 밖으로 옮긴 뒤 다시 시도하는 편이 좋습니다.

---

## 7. 이전 장면 캡처

이전 장면 캡처는 방송을 보다가 순간을 놓쳤을 때 사용합니다.

기본값은 1초이며, 설정에서 0.01초부터 5초까지 조절할 수 있습니다.

사용 방법은 다음과 같습니다.

- 화면의 `-1초` 버튼 클릭
- 확장 프로그램 팝업의 이전 캡처 버튼 클릭
- 기본 단축키 `B` 누르기

확장 프로그램은 재생 중인 영상의 최근 프레임을 일정 시간 동안 임시로 보관합니다. 페이지를 막 열었거나 영상을 방금 재생한 경우에는 캐시가 아직 준비되지 않아 이전 장면을 바로 가져오지 못할 수 있습니다. 이때는 몇 초 정도 재생한 뒤 다시 시도합니다.

캐시에서 원하는 시점을 찾을 수 없지만 플레이어가 해당 위치로 이동할 수 있는 경우에는 재생 위치를 잠깐 이동해 화면을 저장한 뒤 원래 위치로 돌아올 수 있습니다.

### 이전 장면 캐시 프리셋

| 모드 | 특징 | 권장 상황 |
|---|---|---|
| 저부하 | 프레임 수와 크기를 줄여 메모리 사용을 낮춤 | 저사양 PC, 장시간 시청 |
| 균형 | 품질과 자원 사용량의 균형 | 일반적인 사용 |
| 고속 | 더 촘촘하게 프레임을 보관 | 짧은 순간을 세밀하게 잡을 때 |
| 원본 | 영상 크기를 가능한 한 유지 | 이미지 품질이 가장 중요할 때 |

고속과 원본 모드는 메모리 사용량이 늘어날 수 있습니다. 방송을 오래 켜 두거나 여러 탭을 동시에 사용할 때 버벅임이 생기면 균형 또는 저부하로 낮추는 것이 좋습니다.

## 8. 연속 캡처

`연속 캡처`가 켜져 있으면 캡처 단축키를 누르고 있는 동안 여러 장을 연속으로 저장할 수 있습니다.

- `S`를 길게 누르면 현재 장면 연속 캡처
- `B`를 길게 누르면 이전 장면 연속 캡처

키를 처음 누르는 순간 한 장을 저장하고, 설정된 시작 대기 시간이 지나면 반복 캡처를 시작합니다. 탭을 전환하거나 브라우저 창이 비활성화되면 연속 캡처가 중단될 수 있습니다.

### 연속 캡처 간격

설정값이 낮을수록 더 자주 저장하지만 CPU, 메모리, 저장 공간 사용량이 빠르게 늘어납니다.

- 17~50ms: 매우 촘촘함. 짧게 사용할 때만 권장
- 100~250ms: 일반적인 장면 선택에 적당함
- 500ms 이상: 저장 파일 수를 줄이고 싶을 때 적당함

17ms는 초당 약 60회의 캡처를 시도하는 값입니다. 실제 저장 속도는 영상 프레임, 브라우저 성능, 디스크 속도에 따라 낮아질 수 있지만 파일이 매우 많이 생길 수 있으므로 주의해야 합니다.

## 9. 녹화

녹화는 현재 플레이어에서 재생되는 영상을 파일로 저장합니다.

시작 방법:

- 화면의 `녹화` 버튼 클릭
- 확장 프로그램 팝업의 녹화 버튼 클릭
- 기본 단축키 `R` 누르기

녹화 중에는 버튼이 `중지`로 바뀌며 경과 시간이 표시됩니다. 다시 누르면 녹화를 끝내고 파일을 저장합니다.

한 번의 녹화는 최대 5분입니다. 5분이 지나거나 브라우저 메모리 사용량이 안전 한도를 넘으면 자동으로 종료될 수 있습니다.

### 녹화 품질 모드

| 모드 | 기준 비트레이트 | 특징 |
|---|---:|---|
| 저부하 | 약 4Mbps | 자원 사용을 줄이고 호환성을 우선함 |
| 균형 | 약 6Mbps | 화질과 파일 크기의 균형 |
| 화질 우선 | 약 8Mbps | 파일 크기가 늘어나는 대신 높은 비트레이트 사용 |

실제 파일은 브라우저가 지원하는 형식에 따라 MP4 또는 WebM으로 저장됩니다. 녹화 설정은 비트레이트와 코덱 선택에 영향을 주지만, 영상 해상도나 60fps를 강제로 만드는 기능은 아닙니다. 결과 해상도와 프레임은 현재 재생 중인 영상과 브라우저 상태를 따릅니다.

영상이 일시정지돼 있거나 브라우저가 해당 플레이어의 미디어 스트림 녹화를 지원하지 않으면 녹화를 시작할 수 없습니다. 이 경우 영상을 먼저 재생하고 다시 시도합니다.

## 10. 5초 앞뒤 이동

`5초 탐색` 기능이 켜져 있을 때 기본 단축키는 다음과 같습니다.

- `←`: 5초 뒤로 이동
- `→`: 5초 앞으로 이동

라이브 방송에서 앞으로 이동할 수 있는 범위를 넘으면 플레이어가 허용하는 최신 지점까지만 이동합니다. 다시보기에서는 일반적인 5초 탐색으로 동작합니다.

단축키가 다른 웹 기능이나 채팅 입력과 겹치면 설정 화면에서 다른 키로 변경할 수 있습니다.

## 11. 저지연 기능

저지연 기능은 라이브 방송에서만 동작합니다.

브라우저 플레이어가 이동할 수 있는 최신 위치와 현재 재생 위치의 차이를 확인한 뒤, 상황에 따라 재생 속도를 조금 올리거나 최신 위치 근처로 이동합니다. 버퍼가 부족한 상태에서는 보정을 멈추고 잠시 안정화합니다.

저지연 수치는 송출 서버부터 시청자 화면까지의 전체 지연 시간이 아닙니다. 같은 방송이라도 네트워크, 플레이어 버퍼, 화질, 브라우저 상태에 따라 실제 체감 지연은 다를 수 있습니다.

### 저지연 모드

#### 자동

일반 사용자에게 권장하는 기본 모드입니다. 지나치게 라이브 끝에 붙지 않으면서 밀린 재생 위치를 자동으로 따라잡습니다.

- 실제 보정 목표는 대체로 플레이어 최신 위치에서 약 1.15초 이상 뒤
- 최대 보정 속도 약 1.04배
- 필요할 때 높은 화질 선택을 다시 시도할 수 있음

#### 빠른 반응

지연 시간을 더 줄이는 데 우선순위를 둡니다. 네트워크나 PC 상태가 좋지 않으면 버퍼링이 늘어날 수 있습니다.

- 실제 보정 목표는 최소 약 0.75초
- 최대 보정 속도 약 1.06배
- 높은 화질 선택을 다시 시도할 수 있음

#### 안정 우선

라이브 끝에서 조금 더 여유를 두고 버퍼링을 줄이는 데 초점을 둡니다.

- 실제 보정 목표는 대체로 약 1.5초 이상 뒤
- 최대 보정 속도 약 1.04배
- 자동으로 최고 화질을 다시 선택하지 않음

화질을 직접 낮춰 고정했거나 자동·빠른 반응 모드에서 끊김이 생긴다면 안정 우선이 적합합니다.

#### 직접 설정

목표 지연과 확인 간격을 직접 입력합니다. 목표를 0초로 입력해도 실제 동작에는 최소 안전 거리가 적용되므로 완전히 0초에 붙는 기능은 아닙니다.

### 상태 표시

- `저지연`: 목표 범위에서 정상 재생 중
- `저지연↑`: 배속으로 따라잡는 중
- `저지연!`: 버퍼 위험을 감지해 안정화 중
- `저지연Ⅱ`: 사용자가 재생 위치를 옮겨 보정이 잠시 중단됨

사용자가 직접 재생 위치를 움직이면 저지연 보정은 약 10초 동안 쉬었다가 다시 시작합니다. 바로 최신 위치로 돌아가고 싶다면 저지연 버튼을 다시 눌러 상태를 확인합니다.

팝업의 고급 상태에서는 다음 값을 확인할 수 있습니다.

- 플레이어 기준 현재 지연
- 앞으로 재생할 수 있는 버퍼 시간
- 현재 적용 중인 목표
- 재생 속도
- 보정 단계와 마지막 동작
- seek 및 버퍼 위험 발생 횟수

## 12. 통나무 표시 및 자동 수령

라이브 방송에서 `통나무 자동 수령/표시` 기능을 켜면 채팅 영역 근처에 현재 통나무 수량이 표시됩니다.

수령 가능한 통나무 보상이 화면에 나타나면 확장 프로그램이 자동 수령을 시도합니다. 이 기능은 로그인된 CHZZK 계정 상태를 사용하며, 통나무 표시를 누르면 수량을 다시 확인합니다.

자동 수령을 원하지 않으면 다음 경로에서 끕니다.

```text
확장 프로그램 아이콘 → 설정 → 통나무 자동 수령/표시 끄기
```

로그인 상태가 아니거나 CHZZK 측 응답이 늦으면 수량 표시와 자동 수령이 바로 갱신되지 않을 수 있습니다.

## 13. 저장 방식과 폴더 구조

저장 방식은 두 가지입니다.

### 폴더 직접 저장

사용자가 선택한 폴더에 파일을 바로 저장합니다. 브라우저가 폴더 권한을 다시 확인하면 `허용`을 눌러야 합니다.

선택한 폴더 아래에는 방송별 하위 폴더가 만들어집니다.

```text
선택한 폴더/
└─ YYYYMMDD 방송제목/
   ├─ screenshot-....png
   ├─ screenshot-....png
   └─ record-....mp4 또는 webm
```

같은 라이브 방송이 이어지는 동안 제목이 바뀌거나 날짜가 넘어가더라도 같은 방송으로 확인되면 기존 폴더를 계속 사용합니다.

### 브라우저 다운로드

폴더 직접 저장을 사용하지 않거나 브라우저가 해당 기능을 지원하지 않을 때 사용합니다. 파일은 브라우저의 기본 다운로드 설정에 따라 저장됩니다.

폴더 권한이 사라졌거나 저장에 실패하면 브라우저 다운로드 방식으로 전환될 수 있습니다.

## 14. 기본 단축키

| 기능 | 기본 키 |
|---|---|
| 현재 장면 캡처 | `S` |
| 이전 장면 캡처 | `B` |
| 녹화 시작·중지 | `R` |
| 5초 뒤로 | `←` |
| 5초 앞으로 | `→` |

단축키를 바꾸는 방법:

1. 확장 프로그램 설정을 엽니다.
2. 변경할 단축키 버튼을 누릅니다.
3. 사용할 키를 한 번 누릅니다.
4. 단축키를 비우려면 입력 대기 상태에서 `Esc`를 누릅니다.

설정값은 입력 후 바로 저장됩니다.

## 16. 권장 설정

### 일반적인 사용

```text
이전 장면: 1초
캐시 프리셋: 균형
연속 캡처 간격: 100~250ms
연속 캡처 시작 대기: 450ms
저지연 모드: 자동
녹화 모드: 저부하 또는 균형
```

### 이미지 품질 우선

```text
캐시 프리셋: 원본
이전 장면: 0.5~1초
연속 캡처 간격: 100ms 이상
```

원본 캐시는 메모리 사용량이 높아질 수 있으므로 여러 방송 탭을 동시에 켜 두는 환경에는 적합하지 않을 수 있습니다.

### 저사양 PC 또는 장시간 시청

```text
캐시 프리셋: 저부하
연속 캡처 간격: 250ms 이상
녹화 모드: 저부하
저지연 모드: 안정 우선
```

### 지연 최소화 우선

```text
저지연 모드: 빠른 반응
확인 간격: 기본값 유지
```

빠른 반응 모드에서 버퍼링이 생기면 자동 또는 안정 우선으로 낮춥니다. 목표 지연을 더 낮추는 것보다 안정 우선으로 전환하는 편이 실제 시청 경험에는 나을 수 있습니다.

## 17. 자주 생기는 문제

### 화면 버튼이 보이지 않음

1. 현재 주소가 CHZZK 라이브 또는 동영상 페이지인지 확인합니다.
2. 확장 프로그램 팝업에서 `화면 버튼 표시`를 켭니다.
3. CHZZK 탭을 새로고침합니다.
4. `chrome://extensions`에서 확장 프로그램이 켜져 있는지 확인합니다.
5. 업데이트 직후라면 확장 프로그램 카드의 새로고침 버튼을 누릅니다.

### 캡처 파일이 저장되지 않음

1. 브라우저 다운로드 목록에서 차단 또는 실패 여부를 확인합니다.
2. 폴더 직접 저장을 사용 중이면 저장 폴더를 다시 선택합니다.
3. 브라우저가 폴더 권한을 묻는 경우 허용합니다.
4. 영상이 실제로 재생 중인지 확인합니다.

### 이전 장면 캡처가 준비 중이라고 나옴

페이지를 막 열었거나 재생을 시작한 직후에는 이전 프레임이 충분히 쌓이지 않았을 수 있습니다. 설정한 이전 시간보다 조금 더 재생한 뒤 다시 시도합니다.

계속 실패하면 이전 시간을 줄이거나 캐시 시간을 늘립니다. 메모리 부족이 의심되면 캐시 프리셋을 균형 또는 저부하로 바꿉니다.

### 연속 캡처 중 화면이 버벅임

캡처 간격을 늘리고 캐시 프리셋을 낮춥니다. 17ms 설정은 짧은 구간에서만 사용하는 편이 좋습니다. 저장 장치 속도가 느리면 간격을 낮춰도 실제 파일 저장이 따라가지 못할 수 있습니다.

### 녹화가 시작되지 않음

1. 영상을 먼저 재생합니다.
2. 다른 탭이나 플레이어에서 이미 녹화 중인지 확인합니다.
3. 페이지를 새로고침한 뒤 다시 시도합니다.
4. Chrome, Edge, Whale 등 최신 Chromium 브라우저에서 확인합니다.

브라우저가 해당 영상의 `captureStream` 또는 필요한 MediaRecorder 형식을 지원하지 않으면 녹화를 사용할 수 없습니다.

### 녹화 파일이 WebM으로 저장됨

브라우저가 MP4 녹화를 지원하지 않거나 선택한 코덱을 사용할 수 없으면 WebM으로 저장될 수 있습니다. 오류가 아니라 브라우저 호환성에 따른 동작입니다.

### 저지연을 켰는데 숫자가 줄지 않음

다음 상황에서는 보정이 잠시 멈출 수 있습니다.

- 영상이 일시정지됨
- 사용자가 재생 위치를 직접 이동함
- 앞으로 재생할 버퍼가 부족함
- 플레이어가 seek 중임
- 현재 페이지가 라이브 방송이 아님

사용자 이동 뒤에는 약 10초 정도 기다립니다. 버퍼링이 반복되면 빠른 반응 대신 자동 또는 안정 우선으로 변경합니다.

### 화질이 자동으로 높아짐

자동 또는 빠른 반응 모드는 저지연 동작 중 높은 화질 선택을 다시 시도할 수 있습니다. 화질을 직접 낮춰 유지하고 싶다면 안정 우선 모드를 사용합니다.

### 통나무 수량이 갱신되지 않음

CHZZK 로그인 상태를 확인하고 통나무 표시를 눌러 다시 조회합니다. 그래도 표시되지 않으면 페이지를 새로고침합니다. CHZZK 측 API나 화면 구성이 바뀌면 일시적으로 동작하지 않을 수 있습니다.

### 숨긴 버튼을 다시 찾을 수 없음

브라우저 도구 모음의 Chzzk capture 아이콘을 누른 뒤 `화면 버튼 표시`를 켭니다.

## 18. 권한과 데이터 사용

확장 프로그램은 기능 동작을 위해 다음 종류의 권한을 사용합니다.

### CHZZK 라이브·동영상 페이지 접근

화면에 버튼을 추가하고 현재 재생 중인 영상 상태를 확인하는 데 사용합니다.

### CHZZK API 접근

라이브 방송 정보와 통나무 수량 및 수령 상태를 확인하는 데 사용합니다.

### 현재 탭 접근

영상 프레임을 직접 캡처할 수 없을 때 현재 보이는 탭 화면을 이용하는 보조 캡처에 사용합니다.

### 저장소 접근

기능 설정, 단축키, 저지연 상태, 방송별 저장 폴더 정보를 브라우저 안에 보관하는 데 사용합니다.

### 탭 접근

확장 프로그램 팝업에서 현재 열려 있는 CHZZK 탭을 찾고 명령을 전달하는 데 사용합니다.

캡처와 녹화 파일은 사용자가 선택한 로컬 폴더 또는 브라우저 다운로드 폴더에 저장됩니다. 별도의 파일 업로드 기능은 없습니다.

## 19. 기능상 제한

- 한 번의 녹화는 최대 5분입니다.
- 녹화 파일 형식은 브라우저 지원 상태에 따라 MP4 또는 WebM이 됩니다.
- 녹화 해상도와 프레임은 현재 재생 영상과 브라우저를 따르며 강제로 1080p60으로 변환하지 않습니다.
- 이전 장면 캡처는 준비된 캐시와 플레이어가 이동할 수 있는 범위 안에서만 가능합니다.
- 저지연 표시는 서버 전체 지연이 아니라 플레이어 최신 지점 기준입니다.
- 자동·빠른 반응 모드는 높은 화질을 다시 선택할 수 있습니다.
- 연속 캡처 간격을 지나치게 낮추면 CPU, 메모리, 저장 공간 사용량이 크게 늘어납니다.
- CHZZK의 플레이어나 페이지 구조가 바뀌면 일부 기능이 일시적으로 동작하지 않을 수 있습니다.

## 20. 비공식 도구 안내

Chzzk capture는 개인이 제작한 비공식 확장 프로그램이며 NAVER 또는 CHZZK의 공식 제품이 아닙니다.

방송 화면의 캡처와 녹화는 서비스 이용약관, 방송 운영자의 정책, 저작권과 초상권을 지키는 범위에서 사용해야 합니다. 저장한 파일의 사용과 배포에 대한 책임은 사용자에게 있습니다.
